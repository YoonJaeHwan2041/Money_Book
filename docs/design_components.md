# MoneyBook 디자인 컴포넌트 네이밍 가이드

이 문서는 홈/거래 탭에서 재사용하는 주요 UI 컴포넌트의 공통 명칭을 정의한다.

## 1) Big Box

- 의미: 할부금 포함금액/지금 잔고를 보여주는 상단의 큰 요약 카드
- 포함 요소:
  - 메인 금액(할부금 포함금액 또는 지금 잔고)
  - 할부 표시 토글
  - 모드 칩(할부금 포함금액, 지금 잔고)
  - 총 수입/총 지출/총 할부 영역
- 현재 구현 기준 컴포넌트: `TransactionOverviewCard`

## 2) Small Box

- 의미: Big Box 내부의 소형 요약 카드
- 카드 종류:
  - 총 수입
  - 총 지출
  - 총 할부
- 현재 구현 기준 컴포넌트: `OverviewMiniCard` (Big Box 내부)

## 3) Money Box

- 의미: 거래 내역 1건을 나타내는 카드(카테고리/메모/금액/상태)
- 사용 위치:
  - 거래 탭(월별) 거래 리스트
  - 홈 탭 최근 거래 리스트(재사용)
- 현재 구현 기준 컴포넌트: `TradeTransactionRow`

## 명명 규칙

- 문서/디자인 커뮤니케이션에서는 아래 용어를 고정 사용한다.
  - Big Box
  - Small Box
  - Money Box
- 코드 심볼은 기존 Kotlin 네이밍 규칙(파스칼 케이스)을 유지하되,
  PR 설명/이슈/디자인 피드백에서는 위 용어를 우선 사용한다.

## 변경 시 원칙

- 홈/거래 탭에서 동일 디자인이면 반드시 동일 컴포넌트를 재사용한다.
- Big Box 변경 시 Small Box 스타일도 함께 검토한다.
- Money Box 변경 시 홈 최근 거래와 거래 탭 화면을 동시에 확인한다.
